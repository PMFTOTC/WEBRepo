Project Information

We are group of undergraduates from NTU, School of Computer Engineering who have come together to create a interactive and enjoyable web based social network game called NanyangVille. It is adapted from the famous game Farmville.http://en.wikipedia.org/wiki/FarmVille


Gameplay

Once a user is registered, the user is able to create his avatar and would be given a fixed amount of money for him/her to start planting on his farm land. Users are able to plant, fertilise and harvest crops on their own plot of land. One distinctive feature of this game is that the user is able to do the same for his friends and maintain friendship with other users in the game by visiting their farms and presenting gifts.


Market

The market is where the user is able to buy his/her tools are useful for farming. There would be variety of crops and plants to choose from for planting. For some of the items, certain levels have to be unlocked before purchasing using the coins collected.


Project Overview

Goal & objectives

  • To have a better understanding the importance of planning and estimating during project development.
  • To enhance abilities to work as a team
  • To improve software development skills
  • To learn how to develop a game

Stakeholders

  • Guests: They are able to register themselves as a user to play Nanyangville
  • Users: Users are able to have their own account where they are able to log in and out
  • Administrators: They are able to add new functions/ features to the game. They can also manage the user database.

Business requirements

  1. The game must get 50 users each week.
  2. The game must gain a profit 50% by the 3rd quarter since the implementation.
  3. Login frequency should be 20% of the total number of users.
  4. There should be quarterly updates.

Software Requirement Specification

Functional Requirements

3.1 Registration & Login
3.1.1 Users must register using email address first if do not have an account.
3.1.2 The system must send confirmation email to user’s mailbox once register successfully.
3.1.3 Users must login first to play the game.
3.1.4 The system must provide error message if the username and password cannot be matched for login.
3.1.5 The system must pop up a screen showing the reward for login for consecutive days the first time user logs in within one day.

3.2 Farming

3.2.1 Sowing
3.2.1.1 User must have empty farm land to sow a new crop.
3.2.1.2 Once a user chooses a special tool (e.g. a spade) and then clicks on a piece of empty farm land, the shop screen for the seeds must be shown for the user to select from the available virtual crops.
3.2.1.3 The screen must close once a selection is made letting user to sow on empty farm land.
3.2.1.4 The seeds must not be selected if the user does not have enough coins or gems required.
3.2.1.5 The seeds must not be selected if the user does not reach the level required.
3.2.1.6 Upon sowing, coins must be conducted and experience points must be added in user’s account.

3.2.2 Harvesting
3.2.2.1 One-shot virtual crop must experience stages include seed, bush, fully grown with flowers and fully grown with fruits before getting to harvestable stage. The crop withers after being harvest.
3.2.2.2 Re-current virtual crop must experience stages include full grown tree, full grown tree with flowers and full grown tree with fruits before being harvested. After being harvested, the crop is reset to the full grown tree stage.
3.2.2.3 For one-shot crop, user must harvest in certain hours once it reaches the harvestable stage. Otherwise the crop will be withered and coins will be conducted from user’s account.

3.2.3 Killing pests
3.2.3.1 Experience points must be added to user’s account when user kills pests in his own farm or in friend’s farm.
3.2.3.2 The appearance of plot must change with or without the existence of pests.

3.2.4 Fertilization
3.2.4.1 Experience points must be added to user’s account when user fertilizes in his own farm or in friend’s farm.
3.2.4.2 The appearance of plot must change before and after fertilization.

3.2.5 Irrigation
3.2.5.1 Experience points must be added to user’s account when user irrigates in his own farm or in friend’s farm.
3.2.5.2 The appearance of plot must change before and after irrigation.

3.3 Friends system

3.3.1 Search and add friends
3.3.1.1 The system must allow user to search for a friend either using email address or nick name in the game.
3.3.1.2 An add friend request must be sent when user selects a user and clicks add friend.
3.3.1.3 A user must accept, reject or ignore the friend request when a request is received.

3.3.2 Visit friend’s farm
3.3.2.1The system must allow user to visit friend’s farm.
3.3.2 .2The system must allow user to do at most 5 actions every visit.

3.3.3 Send gift to friend
3.3.3.1 The system must pop up a screen showing gifts for selection when user clicks the photo of a friend to send gift.
3.3.4 .1The screen of gifts must close when a gift is selected and confirmed to send.

3.3.4 Send message
3.3.4.1The system must allow a user to leave message on his friend’s farm message board.
3.3.4.2 The message must be shown on user’s message board and can be read and replied by the user.

3.4 Virtual Currencies

3.4.1 The system must provide 50 coins when a user logs in for the first time.
3.4.2 The system must allow a user to purchase coins and gems using real currency.
3.4.3 User must purchase some special items using gems only.

Non-Functional Requirements

4. Non-Functional Requirements

4.1 Interface

4.1.1 The interface must be easily understood by at least 90% of the users.
4.1.2 The web application must display approximately the same interface on all types of browsers.
4.1.3 A user with an average Internet connection must be able to get real time information(eg. level, experience, value of virtual asset) from our server in less than 1 seconds at any time.

4.2 Operation

4.2.1 Accounts
4.2.1.1 The average user must be able to register an account in less than 5 minutes.
4.2.1.2 User’s information must be encrypted and stored in the database.
4.2.1.3 Backups must be made for every user account.
4.2.1.4 Backups of the user’s data must be made while administrator is making changes to the data, but must be deleted when the change is confirmed, to free up memory.

4.2.2 Maintenance
4.2.2.1 The system must inform users 24 hours or earlier should there be a system maintenance.
4.2.2.2 During a maintenance, no users shall be allowed to use the system.
4.2.2.3 The system must backup all data prior to a maintenance.
4.2.2.4 A maintenance must be completed in less than 3 hours.
4.2.2.5 There must not be more than 1 maintenance in 1 month.
4.2.2.6 Changes that do not involve the core functionalities of the system must be able to be implemented without shutting down the system.

4.2.3 Authentication
4.2.3.1 The authentication process must be completed in less than 5 seconds.

4.2.4 Sending Email
4.2.4.1 Email confirmation must be sent in less than 30 seconds when a user registers for an account or when he/she change her password.
4.2.4.2 The system must inform any administrator if a user’s email server fails to respond.

4.2.5 General Operation
4.2.5.1 The system must be able to operate on itself unless in the event of a power failure.
4.2.5.2 The system must restrict the users’ access if there are too many users booking or when they are using up too much memory.
4.2.5.3 The system must be able to finish searching the database for friends, game-related items and etc in less than 5 seconds.

4.3 Failure

4.3.1 In the event of a power failure, the server must be able to recover on its own when power is resupplied.
4.3.2 After the recovery, all unfinished booking must be resumed. Users must be required to log in again.
4.3.3 The system must be able to recover from a “hang” in less than 5 minutes.
4.3.4 If the system is unable to do so, it must alert an administrator.
4.3.5 Before it can recover, the system must block all access from users.

Data Requirements

User account information
1. User has a nickname which should not exceed 100 characters.
2. User's avatar is in a jpg/gif/png format and can be changed at any point using examples stored in a database or own ones.
3. User's profile contains general information status, level and progress of the user in the game.
4. User's password must contain only alphanumeric characters. Password's length is 8 to 20 characters without spacing.
5. User's email address should be in valid form.
6. Quantity of coins and gems (user's balance) is presented as an integer number.
7. Farm control panel contains information about conditions and the amount of time required for crop harvesting.

Farm information
1. "Market" is a collection of flash animated characters and items.
2. Farm's crop has different stages of ripeness: seed, bush, fully grown, withered. Every stage is assigned to a certain type to differentiate.
3. Farm's gifts peresented as a collection of clipart files.
4. Every item has a description which consisting of formatted ASCII text up to 1024 characters in length.
5. Price for a certain item is a non-negative float number.
6. Dates are presented in the format DD/MM/YYYY.
